## ① 변경 요약
- `luida-brain/reflect.rs` — `reflect(conn, cfg, since_ms, factory) -> ReflectReport`
  - 최근 events + 등록 프로젝트 → `learning.reflect`(LLM) → `{patterns, relationships}` JSON
  - 제안 검증(등록 프로젝트 + trigger/action enum) → **비활성(enabled=false)** + `source=learned-promoted`로 upsert
  - `reflected` 이벤트 기록 (원자 tx)
- `luida-cli` — `luida reflect [--since-hours 24]`

## ② 설계 의사결정
- **제안은 비활성 저장**: 학습된 관계는 자동 활성화하지 않음 — 사람이 검토 후 활성화(오작동 방지, spec "propose" 철학). `list_enabled`에 안 나옴.
- **엄격 검증 후 skip**: 미등록 프로젝트·잘못된 trigger/action은 저장 안 하고 skip 카운트 → LLM 환각 차단(DB CHECK + Rust 이중).
- **upsert_by_name**: 같은 이름 제안 재학습 시 갱신(중복 누적 방지).
- **요약함수 아닌 직접 LLM**: reflect는 행위 자체가 LLM 1회 호출 → factory 주입으로 테스트.
