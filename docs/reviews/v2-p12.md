## ② 설계 의사결정
- **자동화 루프 폐쇄**: auto_dispatch가 같은 campaign에 quest를 추가 → run_campaign의 `ready_in_campaign` 루프가 자연히 픽업. 별도 스케줄러 불필요. (원래 동기 "agora 변경 → admin 자동 반영" 실현)
- **활성 관계만 발화**: `list_by_from`이 enabled만 반환 → 학습 제안(비활성)은 사람이 `relationship enable` 전까지 실행 안 됨(안전).
- **propose vs auto_dispatch**: 신뢰도 낮은 건 propose(사람 결정), 높은 건 auto_dispatch. inmail dedupe로 중복 제안 방지.

## ③ 잔여 / 후속
- `path_changed`·`tag_pushed` 트리거(git 변경 감지)는 실환경 watcher 필요 → 후속. 현재 `quest_completed`(in-process) 구현.
- 트리거로 생성된 quest의 무한 연쇄 방지(관계 사이클) 가드는 후속(현재 같은 campaign 내 DAG라 사이클 시 무한 가능 — 단, 관계는 사람 승인 전제라 위험 낮음).
