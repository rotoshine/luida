## ② 설계 의사결정

- **status/kind는 String + DB CHECK + Rust 상수 배열**: enum FromSql/ToSql 보일러플레이트 회피하면서 DB·코드 양쪽 유효성. (v1 TS와 동일 철학)
- **INSERT OR IGNORE의 함정 차단**: enqueue가 OR IGNORE라 UNIQUE뿐 아니라 CHECK도 삼킴 → kind를 Rust에서 명시 검증 (§③ 발견)
- **멱등 insert**: quest.source_inmail_id UNIQUE + insert_idempotent로 dispatch 중복 차단
- **DAG**: quests.depends_on_quest_id + ready_in_campaign이 완료 의존성만 통과시킴
- **project-centric**: v1 adventurers 폐기. quests/relationships가 projects(name) FK
- **agents.json 주석 허용**: `_`로 시작하는 키를 재귀 strip → JSON에 주석 흉내 가능
- **resolve 우선순위**: projectOverrides > actions > defaults, tier 미지정 시 런타임 tier별 모델
