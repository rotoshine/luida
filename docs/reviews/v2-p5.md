## ② 설계 의사결정

- **vault base 주입**: 테스트가 temp dir 사용 → 실제 `~/.luida/memory` 오염 없이 파일 I/O 검증. 런타임 데이터(`~/.luida/`)와 코드 분리 규약 준수.
- **완료 마감 vs 사후 보고 분리**: 전 quest 완료시에만 `completed`. 실패/부분완료도 보고서는 남기되(`set_report_path`) 상태는 유지 → 포스트모템 기록.
- **chronicle는 단일 `chronicle.md`**: 월별 분리(spec `chronicle/2026-05.md`)는 날짜 라이브러리 의존을 피하려 단일 롤링 파일로. 월별 분리는 후속(§잔여).
- **Obsidian frontmatter + wikilink**: 보고서에 `type/campaign_id/status/quests` frontmatter, chronicle에 `[[<파일stem>]]` 링크 → 지식 그래프.
- **fallback 보고서**: LLM이 summary를 안 줘도 quest 목록 기반 기본 보고서 생성 → 항상 기록 보장.
