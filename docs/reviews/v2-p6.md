## ② 설계 의사결정

- **factory = fn 포인터(`cli_factory`)**: `Fn` 구현 → plan/run/report/ingest/resume/triage가 모두 동일 factory로 런타임 해소. 행위별 kind/command는 resolved에 담겨 단일 factory가 분기.
- **CLI 우선(서버 POST보다)**: 로컬 단일 사용자 도구라 CLI가 가장 직접적인 사용 표면. 전 엔진을 즉시 검증 가능. 서버 POST command API·TUI 패널은 후속 UI Phase.
- **ProjectRepo arm별 생성**: ingest의 `&mut conn`과 다른 arm의 `&conn` 차용 충돌 회피 위해 match arm마다 repo 생성(장기 borrow 제거).
- **ingest 견고성**: repo 경로/README 없어도 fallback 노트로 진행 → 등록 직후 비동기 ingest가 실패로 막히지 않음.
